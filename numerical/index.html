🌊 數值模擬

別怕微積分或差分方程——數值模擬其實就是「重複一條簡單的更新規則」。

核心觀念：把時間切成一小步一小步

真實世界是連續變化的，但電腦沒辦法算「連續」。所以數值模擬的做法是：把時間切成很多很小的步（每步 Δt），每一步都用一條簡單規則往前推一次，一直重複。

比喻：就像縮時攝影或一格一格的翻頁動畫。你不是拍下「連續的運動」，而是拍很多張靜態照片，快速播放看起來就連續了。步（照片）越密，看起來越滑順、越準。

整份教學只要記住這一條式子

下一刻的值 ＝ 現在的值 ＋ 變化率 × 一小步 Δt

這就是最基本的歐拉法（Euler method），也是「有限差分」最直覺的樣子。

「差分」到底是什麼？就是把嚇人的微分 dy/dx（瞬間變化）換成相鄰兩點的差 Δy/Δx。一換，微分方程就變成只有加減乘除的算式，電腦一步步算就好。

寫成式子長這樣

「變化率」在數學上就是導數（微分）。差分法把它近似成兩個相鄰點的差：

微分 → 差分近似 dydt ≈ y(t+Δt) − y(t)Δt

把上式左右兩邊乘開、移項，就得到那條「更新規則」的通式（歐拉法，其中 yn 表示第 n 步的值，f 是變化率）：

歐拉法通式 yn+1 = yn + f(tn, yn) · Δt
後面每個例子，都只是把不同的「變化率 f」代進這條通式而已。看的時候可以對照：微分方程（連續世界）↔ 更新式（電腦一步步算）↔ 程式碼。

互動一：讓球掉下來（歐拉法）

自由落體的物理就兩條微分方程：速度的變化率是重力 g、位置的變化率是速度 v。把它們套進歐拉通式，就變成每一步的更新規則：

微分方程
dvdt = g ， dydt = v
更新式（歐拉）
vn+1 = vn + g·Δt ， yn+1 = yn + vn·Δt
精確解（對照用）
y(t) = 12 g t2
程式碼
v = v + g*dt; y = y + v*dt;

下面互動裡的灰球就是用精確解 y = ½gt² 畫的，藍球則是用上面的更新式一步步算——比一比就看得出誤差。

// 每一步（Δt）重複：
速度 = 速度 + 重力 * Δt;   // 速度被重力加速
位置 = 位置 + 速度 * Δt;   // 位置被速度推動

互動：藍球用你選的步長，灰球是「真實答案」

歐拉法（你的步長） 真實位置（精確解）
你會發現：步長越小，藍球越貼近灰球（越準），但要重複算越多次；步長太大，藍球會「衝過頭」——這就是數值誤差。這是所有模擬的核心取捨：精準度 vs 計算量。

互動二：咖啡慢慢變涼（牛頓冷卻）

一杯熱咖啡放著會慢慢降到室溫，而且離室溫越遠、降得越快。「降溫速度正比於溫差」寫成微分方程就是牛頓冷卻定律，再套歐拉通式變成更新式：

微分方程
dTdt = −k(T − T室)
更新式（歐拉）
Tn+1 = Tn − k(Tn − T室)·Δt
精確解
T(t) = T室 + (T0 − T室) e−k t
程式碼
T = T + (-k)*(T - 25)*dt; // 室溫 25

精確解裡的 e−kt 就是「指數衰減」——這也是為什麼曲線會先陡後平、慢慢貼近室溫。

互動：從 90 度開始冷卻

曲線一開始陡（溫差大、降得快），越接近室溫越平緩——這種「指數趨近」在自然界到處都是（充放電、藥物代謝、人口趨於上限…）。你只用一條加法規則就模擬出來了。

互動三：熱在鐵棒上擴散（空間的差分）

前兩個例子只跟「時間」有關。這個例子多了「空間」：一根鐵棒，中間一段很燙，熱會慢慢往兩邊散開、最後變均勻。把鐵棒切成一格一格，每一格的更新規則是——往「左右鄰居的平均」靠近一點：

偏微分方程
∂T∂t = α ∂2T∂x2 （熱傳導方程）
空間二階差分
∂2T∂x2 ≈ Ti−1 − 2Ti + Ti+1Δx2
更新式
Tin+1 = Tin + α(Ti−1n − 2Tin + Ti+1n)
程式碼
next[i] = T[i] + a*(T[i-1] + T[i+1] - 2*T[i]);

那條粉紅色的 Ti−1 − 2Ti + Ti+1 就是「二階差分」，數學上對應 ∂²T/∂x²（拉普拉斯項）。它的意思很直白：比左右鄰居的平均低就升溫、高就降溫，所以熱會慢慢抹平。

📈 先搞懂二階導數：從「一階」爬到「二階」

一句話串起來：值 →（取變化率）一階導數 →（再取一次變化率）二階導數。最有感的版本就是 位置 →（斜率）速度 →（斜率的斜率）加速度。拉滑桿讓游標沿著曲線走，看這三張圖怎麼連動：

三張圖的連動關係：②的高度＝①那條白色切線有多斜；③的高度＝②正在往上爬還是往下掉。當②衝到最高（斜率最大）時，③剛好穿過 0；③為正的地方，①就凹朝上 ∪；③為負，①凹朝下 ∩。這就是「一階＝斜率、二階＝斜率的斜率」最直接的畫面——不是背的，是看得到的。

💡 為什麼剛好是「左 + 右 − 2×自己」？（不用背，用想的）

這條式子不是硬湊的。二階導數的白話就是「你彎不彎」——而電腦手上只有幾顆點，量彎曲最省事的辦法就是問一句：「我，比左右鄰居的平均，高還是低？」

把式子重排就看得出來：

T左 − 2T中 + T右 = 2 × [ T左 + T右2 − T中 ]

括號裡就是「鄰居平均 − 我」。所以：低於鄰居平均 → 正 → 升溫；高於 → 負 → 降溫；剛好落在鄰居連線上（直線、不彎）→ 0 → 不變。

那中間的「−2」到底怎麼來的？把「斜率的斜率」硬算一遍就露餡了。先寫出「我」左右兩段的坡度，再相減：

① 右斜率
(T右 − T中) / Δx　我是這段起點 → 被減（−T中）
② 左斜率
(T中 − T左) / Δx　我是這段終點 → 是加的（+T中）
③ 二階 ＝ ①−②
(T右 − T中) − (T中 − T左)
④ 拆掉括號
= T右 − T中 − T中 + T左　②前面是減號，+T中 翻成 −T中
⑤ 把「我」收在一起
= T左 − 2T中 + T右

關鍵在第 ④ 步：「我」被扣了兩次 −1——第一次在右斜率裡當起點（−T中）；第二次是左斜率本來 +T中，卻被「③ 減掉左斜率」那個減號翻成 −T中。兩個 −1 疊起來就是 −2。而左鄰、右鄰各只出現一次、都是 +1，所以最後長成 T左 − 2T中 + T右。（兩段坡度各除一次 Δx、最後再除一次，就湊成式子裡的 Δx²。）

嚴謹版（進階）：把左右兩點用泰勒展開在中間點附近展開再相加，帶奇數次方的誤差項（一階、三階…）會一正一負互相抵消，乾淨留下二階導數。這就是為什麼要「左右對稱」取點——對稱才消得掉誤差，也讓這條式子特別準。所以無論從直覺（離鄰居平均多遠）還是數學（泰勒相消）出發，都指向同一條式子，真的不用背。
// 每一步，每一格 i：
新溫度[i] = 溫度[i] + α * (溫度[i-1] + 溫度[i+1] - 2*溫度[i]);

那條 左 + 右 - 2×自己 就是二階差分（數學上叫拉普拉斯項），也就是「熱傳導方程」的差分版。看起來嚇人的偏微分方程，落到程式裡就是這一行加減乘除。⚠️ 把 α 拉太大，數值會「爆掉」亂跳——這叫數值不穩定，提醒你步長不能貪心。

互動四：熱在平面上擴散（二維）

把上一個例子從「一根鐵棒」升級成「一整片鐵板」。差別只有一個：一維時每一格只有左、右兩個鄰居；二維時每一格有上、下、左、右四個鄰居。規則還是同一句話——往鄰居的平均靠近一點。

比喻：一維像排隊的人，只跟前後兩位借體溫；二維像坐在方格座位上，跟前後左右四個人一起取平均。鄰居多了兩個，熱散得更快、也更均勻。
偏微分方程
∂T∂t = α(∂2T∂x2 + ∂2T∂y2)
空間差分（五點）
T上 + T下 + T左 + T右 − 4T中
更新式
Ti,jn+1 = Ti,jn + α(Ti−1,j + Ti+1,j + Ti,j−1 + Ti,j+1 − 4Ti,j)
程式碼
next[i][j] = T[i][j] + a*(T[i-1][j]+T[i+1][j]+T[i][j-1]+T[i][j+1] - 4*T[i][j]);

看出規律了嗎？一維是「左 ＋ 右 − 2×自己」，二維就是「上 ＋ 下 ＋ 左 ＋ 右 − 4×自己」。多一個維度，就多兩個鄰居，中間的係數也從 2 變成 4。這個 T上+T下+T左+T右−4T中 叫做「五點差分」，是二維拉普拉斯項 ∇²T 的離散版。這裡的 i 是「第幾列」、j 是「第幾行」，n 一樣是「第幾步（時間）」。

💡 那二維的「−4×自己」是怎麼來的？（就是 x 的彎 ＋ y 的彎）

二維的「彎不彎」很單純——把 x 方向的彎和 y 方向的彎加起來，這就是拉普拉斯：∇²T = ∂²T/∂x² + ∂²T/∂y²。各自套上一維那條二階差分（左−2我+右）：

(左−2我+右) + (上−2我+下) = 上+下+左+右 − 4我

那個 −4 一點都不神祕：橫向扣 2、縱向扣 2，加起來就是 −4。重排後意義跟一維一模一樣，只是鄰居從 2 個變 4 個：上+下+左+右 − 4我 = 4 ×〔(4 個鄰居的平均) − 我〕。拉中間那格的溫度看看它是升是降：

小結：模擬其實不難

回頭看，這幾個模擬都長一樣：

  1. 選一條更新規則（下一刻＝現在＋變化×Δt）。
  2. 用迴圈重複它很多次。
  3. 把每一步畫出來，就成了動畫。

建議的學法：先別急著看嚴謹推導。先動手跑、看動畫、拉拉步長，建立「一小步一小步往前推」的直覺；等你對它有感覺了，再回頭看課本上的微分方程，會發現它們只是把你剛剛做的事寫成緊湊的符號而已。

下一步（進階）：步長太大會不準或不穩定，這時可以用更小的步長，或更聰明的方法（如 RK4 龍格–庫塔）在同樣步長下更準。但那些都是「同一套心法」的升級版——核心永遠是那條更新規則。
