🧭 演算法

「事情怎麼做才快」——用圖解與互動動畫理解核心演算法觀念。

時間複雜度 Big-O

Big-O 描述當資料量 n 變大時，「工作量」成長得多快。我們只在乎成長「趨勢」，不在乎精確秒數。

比喻：找一本書。書架有標籤（照字母排好）你可以「對半找」＝很快；書堆成一團只能一本本翻＝很慢。資料越多，這個差距越誇張。

常見等級（由快到慢）

O(1) 常數 O(log n) 對數 O(n) 線性 O(n log n) O(n²) 平方
記號名稱白話例子
O(1)常數不管多少資料，一步搞定用索引拿陣列的某一格
O(log n)對數每一步砍掉一半二分搜尋
O(n)線性每筆都看一次從頭找到尾
O(n log n)線性對數好的排序法極限合併／快速排序
O(n²)平方雙層迴圈，資料翻倍時間變四倍氣泡排序
記一句話就好：n 變大時，能不能少看一點資料？能對半砍就是 log，能只掃一次就是 n，要巢狀比較就會變 n²。

遞迴 Recursion

遞迴就是「函式呼叫自己」，把大問題切成一模一樣的小問題，直到小到不能再切（終止條件）。

比喻：兩面鏡子對照會出現無限層倒影。遞迴也是一層包一層，但一定要有「停下來」的那一層，否則會無限下去（程式會爆掉）。
// 階乘 5! = 5 × 4 × 3 × 2 × 1
function factorial(n) {
  if (n <= 1) return 1;   // ← 終止條件（最小問題）
  return n * factorial(n - 1); // ← 呼叫自己，問題縮小
}
factorial(5); // 120

展開過程（點按觀察）

寫遞迴的口訣：①先寫終止條件 ②再讓問題「變小」呼叫自己。缺了終止條件就是無限迴圈。

氣泡排序 Bubble Sort

相鄰兩個比一比，比較大的往後「冒泡」，一輪把最大的推到最後。簡單但慢，O(n²)。

更快的排序：合併與快速

實務上用 O(n log n) 的排序，關鍵是「分而治之」——把資料切一半、各自排好、再合起來。

合併排序 Merge Sort

一直對半切到剩單個，再兩兩「合併」成有序。穩定、最壞也是 O(n log n)。

快速排序 Quick Sort

選一個「基準 pivot」，比它小的丟左邊、大的丟右邊，再各自遞迴。平均最快。

二分搜尋 Binary Search

前提：資料已經排好序。每次都看「正中間」，比目標大就往左半找、比目標小就往右半找，每一步砍掉一半 ➜ O(log n)。

比喻：猜 1～100 的數字。與其一個個猜，不如先猜 50——「太大」就往 1～49 猜，「太小」就往 51～100。最多七次就中。

互動：在 1～20 裡找一個數

function binarySearch(arr, target){
  let lo = 0, hi = arr.length - 1;
  while (lo <= hi){
    const mid = Math.floor((lo + hi) / 2);
    if (arr[mid] === target) return mid;
    if (arr[mid] < target) lo = mid + 1;
    else hi = mid - 1;
  }
  return -1;
}

圖的走訪 BFS / DFS

圖 (graph) 是節點 + 連線，可以任意相連（像地圖、社群網路）。要「走遍」所有節點有兩種經典方式。

互動：從節點 A 出發走訪

BFS 廣度優先 · 用佇列

像水波紋一圈圈擴散：先走完所有「鄰居」，再走鄰居的鄰居。常用於找最短路徑。

DFS 深度優先 · 用堆疊/遞迴

像走迷宮：一條路走到底、走不通再回頭換路。常用於探索所有可能、判斷連通。

一句話記憶：BFS＝「先看近的」（佇列，先進先出）；DFS＝「一路衝到底」（堆疊，後進先出）。
