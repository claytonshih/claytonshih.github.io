🗃️ 資料結構

「資料怎麼擺」——擺對地方，找東西、放東西才會快。

陣列 Array

最基本的結構：一排連續的格子，每格有一個索引（index），從 0 開始。知道索引就能瞬間拿到值。

01234
12
7
9
4
15
比喻：像一排編號的信箱。要拿 3 號信箱的信，直接走過去開＝O(1)；但若要在中間「插一格」，後面全部信箱都得往後搬＝O(n)。
操作複雜度為什麼
用索引讀取 arr[i]O(1)直接算出位置
尾端加入 pushO(1)放最後即可
中間插入／刪除O(n)後面元素要整批位移
搜尋某個值O(n)可能要看完整排
const arr = [12, 7, 9, 4, 15];
arr[2];          // 9  ← 直接取，O(1)
arr.push(20);   // 尾端加入，O(1)
arr.splice(1, 0, 99); // 在索引 1 插入 99，O(n)

鏈結串列 Linked List

每個節點（node）存資料＋一個指向下一個節點的箭頭。節點在記憶體不必連續，靠箭頭串起來。

比喻：像尋寶遊戲的紙條——每張紙條寫著寶物＋「下一張在哪」。想找第 4 張，只能從第一張開始一路跟著箭頭走。

✅ 優點

頭尾插入／刪除只要改箭頭，O(1)，不必像陣列整批搬移。

⚠️ 缺點

不能用索引直接跳，要找第 i 個得從頭走，O(n)；且多花記憶體存箭頭。

class Node {
  constructor(value) {
    this.value = value;
    this.next = null;  // 指向下一個節點
  }
}
const a = new Node(12);
a.next = new Node(7); // 12 → 7

堆疊 Stack 與 佇列 Queue

兩者都是「限制進出方式」的串列。差別只在——從哪一端拿東西。

堆疊 Stack · 後進先出 (LIFO)

比喻：一疊盤子。最後放上去的最先被拿走。

佇列 Queue · 先進先出 (FIFO)

比喻：排隊買飲料。先排先買，從隊伍前面出。

用途舉例：堆疊 ➜ 瀏覽器「上一頁」、程式的呼叫堆疊、括號比對；佇列 ➜ 印表工作排程、BFS 走訪。

雜湊表 Hash Table

又叫「字典」或「map」。用一個鍵（key）直接對應到值（value），查找幾乎是 O(1)。

比喻：像通訊錄——你不必一頁頁翻，直接用「名字」就查到電話。背後靠一個「雜湊函式」把 key 算成一個位置，直接跳過去拿。
const stock = new Map();
stock.set("apple", 5);   // 放入 key→value
stock.get("apple");      // 5  ← 幾乎瞬間，O(1)
stock.has("banana");    // false

這裡的雜湊函式：把 key 每個字母的碼加起來，再 % 8，得到「該去第幾個桶」。同一個桶就用串接掛在一起——這就是「碰撞」。

二元樹與二元搜尋樹 BST

樹是「一對多」的分層結構：一個根（root），每個節點往下長出子節點。二元樹每個節點最多兩個小孩（左、右）。

比喻：家譜或公司組織圖——一個人（節點）底下有下屬，層層往下分。

二元搜尋樹 (BST) 多一條規則：左邊子樹全部比自己小、右邊全部比自己大。因此找一個數時，每一步就能決定往左或往右，跟二分搜尋一樣快 O(log n)（樹平衡時）。

上圖就是一棵 BST：以根 8 為例，左邊 3、1、6 都 < 8，右邊 10、14 都 > 8。要找 6？8→往左到 3→往右到 6，只三步。想看跟它一樣快的搜尋法，讀 二分搜尋。
function searchBST(node, target){
  if (!node) return false;
  if (node.value === target) return true;
  return target < node.value
    ? searchBST(node.left, target)   // 比較小 → 往左
    : searchBST(node.right, target); // 比較大 → 往右
}

互動：自己種一棵 BST、看搜尋怎麼走

插入時從根往下比：比較小往左、比較大往右，找到空位掛上去。搜尋走同一條路——所以每一步都能砍掉一半，很快。
