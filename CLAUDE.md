# CLAUDE.md — 直覺科學互動學習（easy_learning）

## 專案概述
純前端教學網站，用生活比喻、圖解與可操作動畫，把複雜科學講到直覺好懂。線上位置 https://claytonshih.github.io/ ，原始檔即本資料夾。技術為 HTML / CSS / vanilla JavaScript，每頁多半有 SVG 圖解或互動元件（滑桿、按鈕、動畫），深色主題、可離線開啟。內容兩條線：資訊科學（演算法、資料結構、設計模式）與計算物理（有限差分、數值模擬、波動模擬、流體力學、水波理論）。共用樣式在 `assets/style.css`。

## 內容編輯原則（重要）

1. **前後連貫、能獨立讀懂。** 每一頁都要像寫給第一次來的讀者，從頭到尾自成一體。任何新增或修改的段落，都必須和前後文自然銜接（有過渡、順序合理），不是硬插進去。改完要通讀整頁一次，確認節奏一致。

2. **內容不得寫成在「回應提問」。** 頁面正文是給讀者的教材，不是對話回覆。禁止留下聊天痕跡或反應式語氣，例如「你剛剛問的」「這一步跳太快」「真正跳的是②」「憑什麼是…？」之類。要帶出「為什麼」時，用課本式、中性的自問自答（例如「這一步拆開看就清楚了」），而不是回嘴口吻。

3. **維持既有教學風格。** 節奏一律是：生活比喻 → 圖解 → 可操作互動。沿用 `assets/style.css` 既有的元件類別（`.card`、`.steps`、`.eq`、`.tip`、`.analogy` 等）與深色配色變數，不要另立風格。

4. **跨頁連結要以讀者為中心。** 指向其他頁時，寫成對讀者有用的前後導引（「想先打好地基，可先讀 X」「延伸請看 Y」），而不是描述我們在對話裡做過的事。

## 版面慣例
- 每頁 `topnav` 導覽列順序：首頁、演算法、資料結構、設計模式、有限差分、數值模擬、波動模擬、流體力學、水波理論。新增頁面時要同步更新所有頁的導覽列與 `sitemap.xml`。
- 區塊 `div`（`.tip`、`.analogy` 等）要用 `</div>` 正確收尾，不要用 `</p>` 代收。
